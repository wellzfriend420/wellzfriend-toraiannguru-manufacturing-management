# 入力画面・食品加工管理帳票対応設計

更新日: 2026-07-19  
状態: 最新仕様・正本（確認事項を除く）

## 基本方針

入力画面は「食品加工管理帳票.xlsx」を業務フロー、項目名称、項目順、確認順の正本とする。Excelの横長表は再現せず、スマホでは1件を縦型フォームまたはカードとして表示する。上下スクロールのみとし、横スクロールを発生させない。

優先順位は、実際の現場運用、既存管理帳票、ダッシュボード表示要件、DB設計の順とする。DB都合で入力順を変更しない。帳票にない項目を追加する場合は、理由と利用目的を提示して承認後に反映する。

## 検品票 → 仕入・入荷入力

原本: `①検品票!A1:I12`

スマホ入力順は次の順序を維持する。

1. 受入日
2. 仕入先
3. 品目
4. 箱数
5. 担当者
6. 納品数量
7. 実測数量
8. 破棄数量
9. 備考

1仕入れにつき1件として登録する。数量・重量の単位、状態確認項目は実運用確認後に確定し、帳票にない項目を推測で追加しない。

## 加工管理帳票 → 工程実績入力

対象原本:

- `②れんこん水煮加工表!A5:H5`
- `③れんこんチップス加工表!A5:H5`
- `⑤ねぎ加工表!A5:H5`
- `⑥きゅうり加工表!A5:H5`

基本入力順は次の順序を維持する。

1. 作業日
2. 使用材料（入荷日）または使用水煮（加工日）
3. 使用量
4. 破棄
5. 出来高
6. 担当者
7. 備考

帳票の「時間（分）」は通常フォームから除外する。Phase1ではLINEの開始・終了から取得し、LINE連携前または例外時のみ、理由を伴う管理者用の例外入力として別画面で扱う。

1行＝1人＝1作業という原本単位を維持する。使用量、破棄、出来高はgで保存・表示する。

## チップス袋詰め加工表

原本: `④チップス袋詰め加工表!A5:J5`

入力順は、作業日、使用チップス（加工日）、11g、40g、500g、その他、破棄、担当者、備考とする。時間（分）の扱いは他の加工帳票と同じとする。11g、40g、500g、その他は袋数で扱う。

## 現行実装との差分

|項目|現行実装|帳票基準との差分|対応状態|
|---|---|---|---|
|仕入・入荷入力画面|未実装|検品票9項目の画面が必要|未実装|
|作業日|実装済み|一致|継承|
|使用材料・入荷日|未実装|ロット／日付選択が必要|修正対象|
|使用量|「投入量」として実装|帳票名称を「使用量」へ合わせる必要|修正対象|
|破棄|「廃棄量」として実装|帳票名称を「破棄」へ合わせる必要|修正対象|
|出来高|「加工後量」として実装|同一値と確定。画面名称を「出来高」へ変更|修正対象|
|担当者|未実装|1人1作業の担当者選択が必要|修正対象|
|工程|選択式で実装|フォーム内の工程選択を残す|確定|
|商品|青果のみ選択式で実装|商品別ダッシュボード集計のために追加承認済み|継承|
|時間|通常入力なし|LINE取得方針と一致|継承|
|備考|実装済み|一致|継承|

## 確定事項（2026-07-19）

1. 「加工後量」と帳票の「出来高」は同一値とする。画面では帳票名称の「出来高」を使用する。
2. フォーム内の工程選択を残す。
3. 検品票の数量単位は品目によって異なり、品目マスタの管理単位に従う。
4. LINE連携前・例外時の時間入力をPhase1から専用画面として用意する。通常加工フォームには時間欄を置かず、Phase2のLINE連携は同じ工数データへ接続する。
