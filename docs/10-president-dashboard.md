# ⑩ ダッシュボード設計

ダッシュボードは実績を確認する参照画面です。工程実績、在庫、製造、出荷などの入力画面はダッシュボード内に設置せず、三本線メニューから開く独立した管理入力画面として提供します。ログインや閲覧専用アカウントによる制限は設けません。

## 目的

社長が朝、前日の製造結果を短時間で把握し、今日の入荷・加工・製造・人員の段取りを決められることを受入基準とします。トップは判断用、詳細は原因確認用に分け、入力機能は置きません。

## 画面骨格

一般的なBIの大型KPIカード列ではなく、確定した工程順を中心にします。Phase1のれんこんトップは「水煮加工」「チップス加工」と、その終点である「完成品」「出荷」を表示します。納品準備と配達・納品は詳細へ置きます。

```text
┌──────────────────────────────────────────────┐
│ [れんこん] [青果]  [日次][週次][月次]  対象日 2026/07/17 │
│ 最終確認 07/18 07:10・確認済み                         │
└──────────────────────────────────────────────┘

入荷実績／加工投入
  原体 500,000g
      │  ※先頭工程投入量と同じ場合は一度だけ表示
      ▼
┌ 水煮加工 ─────────────────────────────────┐
│ 投入 500,000g → 加工後 420,000g ／ 廃棄 30,000g ／ 歩留まり 84% │
└──────────────────────────────────────────┘
      ▼
┌ チップス加工 ───────────────────────────────┐
│ 投入 420,000g → 出来高 100,000g ／ 廃棄 12,000g ／ 歩留まり 24% │
└──────────────────────────────────────────┘
      ▼
完成  完成重量 100,000g ／ 完成数量 2,400袋

完成品在庫（商品マスタ単位）        フレーバー在庫（味別袋数）
40g 塩          1,200袋             塩         35袋
40g コンソメ      800袋             コンソメ   18袋
11g プレーン      900袋             その他     10袋
[在庫詳細を見る]

対象日の出荷売上合計  150,000円
[商品別採算を見る]

[工程分析] [人員・個人実績]
```

## 表示順

1. 部門・期間・確定状態
2. 入荷実績／加工投入量から始まる工程フロー
3. フロー終点の完成重量・完成数量
4. 商品マスタ単位の完成品在庫袋数
5. 味別フレーバー在庫袋数
6. 対象日の出荷売上合計と商品別採算詳細への導線
7. 工程分析、人員・個人実績、在庫詳細への導線

## トップ画面に残す判断項目

1. 工程フローの始点としての加工投入量
2. 工程フロー上の工程別投入量・加工後量・廃棄量・歩留まり
3. 工程フローの終点としての完成重量・完成数量
4. 商品マスタ基準の完成品在庫
5. 味別フレーバー在庫
6. 対象日の出荷売上合計

総合歩留まりを単独KPIとしてトップ表示しません。工程ごとのロスを優先して確認できる構成にします。

## 詳細画面

|詳細|表示内容|
|---|---|
|工程分析|工程別投入量、加工後量、廃棄量、歩留まり、工程時間、工程別労務費、工程別生産性、納品準備時間、配達時間|
|人員・個人実績|担当者、担当工程、個人使用量、個人出来高、個人破棄量、人別歩留まり、LINE／工程入力作業時間、生産性|
|商品別採算|商品別売上、SKU直接原価、粗利益、粗利率、対象日売上合計。チップス共通労務費は別枠表示|
|在庫詳細|原料、加工済みれんこん、完成品、フレーバー、包材|

人別歩留まりは工程分析の担当者欄へ重複表示せず、人員分析で一元表示します。

## トップ画面で表示しないもの

- 原料在庫の大きなサマリー
- 単独の総合歩留まりカード
- 加工投入量と同一値の先頭工程投入量の重複カード
- 個人別歩留まり一覧
- 商品別の売上・原価・粗利・粗利率の全展開
- 出荷予定との差額や在庫不足予測（Phase1対象外）
- 帳票と既存工程マスタで確定していない工程名

## 青果トップ

青果タブは在庫と取引先別売上を表示せず、「きゅうり」「ねぎ」の作物ごとに次の項目を縦方向へ表示します。

```text
きゅうり
収穫量g → 選別後量g → 廃棄量g → 歩留まり% → 出荷量g → 売上円

ねぎ
収穫量g → 選別後量g → 廃棄量g → 歩留まり% → 出荷量g → 売上円
```

青果歩留まりは `選別後量 ÷ 収穫量 × 100` とします。収穫量が0の場合は算出不能とし、0%にはしません。

基本の閲覧専用、日・週・月切替、原本再計算、詳細への遷移はれんこんタブと共通です。

## 操作

- 部門タブと期間ボタンはページ遷移せず再集計
- 前後ボタンで前日／前週／前月へ移動
- KPI・棒グラフ・表の行を押すと根拠明細を表示
- 社長の既定表示は「前日・れんこん」。設定で変更可
- データ0と未入力を区別し、未入力は「未入力」と表示
- 画面内に「本日製造」「本日納品」を使わず、表示対象日と「製造実績」「納品／出荷実績」を使用
- Dashboardと集計表は直接編集不可。訂正は原本入力画面から行い、表示値を再計算

## スマホ表示ルール

- スマホは上下スクロールだけを許可し、横スクロールを発生させない
- 横長表はカード、詳細画面、折りたたみ、縦並びのいずれかへ変換する
- 工程分析は工程ごとのカードとし、工程名、投入量、加工後量、廃棄量、歩留まり、作業時間を縦方向に表示する
- 人員分析と商品別採算もスマホでは横長表を使用せずカード表示とする
- 開いた直後に重要情報が把握できる情報順を優先する

## 色と警告

- 通常値は落ち着いた中性色、警告だけ黄・赤
- 歩留まりの良否は工程別目標値が確定するまで色評価しない
- 在庫下限は品目マスタ設定後に警告。初期は負在庫・未入力のみ警告
- 金額は円、重量はg、製品は袋／個など商品単位で表示。kgへ自動変換しない

## 瞬時切替の性能目標

- 通常データ量で部門・期間切替のAPI応答95%を1秒以内
- 初回表示2秒以内を目標
- 同じ条件の再表示は短時間キャッシュ可。ただし締め・訂正時に破棄

## 受入確認事項

- 手書きメモ上段の時間帯別数値が入荷か作業か
- 売上は出荷基準か請求基準か
- 直接原価のロット評価、端数処理、外注費配賦方法
- 週の開始曜日
- 前日が休業日の場合に直近稼働日を出すか
- 社長が人別指標を評価用途に使うか、段取り用途だけか
