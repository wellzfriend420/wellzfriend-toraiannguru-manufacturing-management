# ⑨ 集計ロジック一覧

## Phase2A工数

- LINE打刻: 終了－開始－休憩打刻合計
- 固定休憩: 各固定休憩帯と作業区間の重複分だけ控除。複数帯・日またぎ対応
- 管理者訂正: 訂正後の開始・終了・休憩分で再計算し、理由と訂正前後を保存
- 加工実績紐付け候補: 作業日・担当者・工程が一致する完了済みLINE工数

## 原本と派生値

現場実績、個人別加工実績、工程実績、入出庫、製造、出荷を原本とします。Dashboardと集計表は原本から算出する派生値であり、直接入力・直接編集を禁止します。訂正後は対象期間を再計算します。

## 共通期間

- 日次: anchorの業務日1日
- 週次: 月曜日から日曜日の7日間
- 月次: anchorを含む暦月
- すべてJST業務日で抽出し、取消済みを除外する
- 数量は基準単位へ換算後に合計し、表示時だけ丸める

青果は収穫量を算出しない。商品別の検品実測数量は検品原本、加工使用量・出来高・加工時破棄は加工帳票原本から集計し、加工歩留まりを`Σ出来高 ÷ Σ使用量`で算出する。検品時破棄は仕入品質指標、加工時破棄は加工工程指標として別集計する。

## Dashboard表示抑制ルール

- 加工投入量と先頭工程投入量が同じ原本・同じ数量なら一度だけ表示する。
- 完成重量・完成数量は工程フロー終点に表示し、独立カードへ重複表示しない。
- 総合歩留まりは算出・表示せず、工程別歩留まりだけをフロー内に表示する。
- 人別歩留まりは個人実績詳細の一箇所だけに表示する。
- 原料在庫は原本として保持するがトップ主要サマリーから除外する。
- 商品別採算はトップでは出荷売上合計だけとし、商品別値は詳細APIで返す。
- 完成品在庫は商品マスタ単位の袋数、フレーバー在庫は味別袋数を基本表示する。

## 計算式

```text
工程ロス量 = max(投入基準数量 - 産出基準数量 - 明示した副産物数量, 0)
工程歩留まり率 = 産出基準数量合計 / 投入基準数量合計 * 100
廃棄率 = 廃棄基準数量 / 投入基準数量 * 100
人別生産性(g/h) = 担当作業量g / (担当時間分 / 60)
売上 = Σ(数量 × 確定単価) または理由付き確定金額
在庫残 = Σ(指定日時点以前のquantity_delta_base)
```

既存ロジックの単純平均は採用せず、期間歩留まりは工程ごとに投入・産出の合計から算出します。異なる工程を一つの歩留まりに合算すると二重計上になるため、Dashboardトップでは総合歩留まりを単独表示せず、工程別に表示します。

## 人別歩留まりの確定仕様

実運用の個人別加工表を原本として次の方法で集計します。

- 対象: 個人別加工実績が登録された工程
- 投入: 個人行の使用量g
- 完成: 個人行の出来高g
- 廃棄: 個人行の破棄g
- 複数人: 1人1行で実績を分け、工程全体は個人行を合計
- 期間: Phase1の日・週・月で同一式を使用
- 訂正: 個人行の取消・訂正版から常に再計算
- 表示: 「人別歩留まり」とし、「段取り・生産状況確認用。人事評価用ではない」と注記

## 労務費配賦

```text
工程別作業時間 = Σ 有効作業セッション分
工程別労務費 = Σ（適用時間単価 × 作業分 ÷ 60）
```

時給者は登録時間単価を適用します。月給者は次の管理単価を使用します。

```text
月給者の管理時間単価 = 月給 ÷ 当月平日日数 ÷ 8
```

これは原価配賦専用で、給与計算には使用しません。チップス加工までの労務費はチップス全体の製造原価単位へ集約し、フレーバー別・内容量別・SKU別には配賦しません。

商品別採算ではSKUへ直接対応できる原料・フレーバー・包材・外注費を商品別直接原価として扱い、チップス共通労務費は商品別粗利へ重複配賦せず「チップス全体共通労務費」として別表示します。

## チップス半製品

規格数量にならなかった端数は廃棄量へ含めず、チップス半製品在庫への入庫として記録します。次回袋詰め時は同在庫からの出庫とし、期間をまたいでも原本を追跡します。

## 売上・売掛

- 売上集計日は納品日。
- 現金販売は納品日売上と同日入金済みを記録。
- 売掛販売は納品日売上と売掛発生を記録し、後日の入金消込で残高を減額。
- 入金予定日は売掛ごとに保持し、資金繰り連携へ渡す。

人別時間・人別工程時間はLINE開始・終了から得ます。LINE連携後は加工実績表から時間列を削除し、元セッションと算出分数をDBに保持します。人別生産性は出来高とLINE時間から計算します。未入力を0としてランキングしません。

## 日次締めと訂正

未確認データも集計しますが、ダッシュボードに状態を表示します。締め後の訂正は旧取引を取消し、訂正版を関連付け、全指標を原本から再集計します。派生値への差分加算・手修正は禁止します。

## 損益（Phase2）

- 部門売上 = 部門別売上明細合計
- 材料費 = 消費ロット数量×ロット単価
- 包材・フレーバー費 = 製造消費明細×使用ロット単価
- 労務費 = 工程別時間×適用時間単価。チップスは全体原価プールへ集約しSKU別配賦なし

## 原料・外注費

- 直接使用原料は使用ロット数量g×ロット単価を直接原価へ計上する。
- 原料ロットの仕入金額は納品書に基づく管理者入力を正本とし、検品明細・原料ロットへ紐付けて使用する。仕入金額未入力のロットは原価確定済みにしない。
- 完成品ロットへ、消費した原料・フレーバー・包材・直接資材の単位原価内訳を引き継ぐ。工程歩留まりによる原価増加は出力数量当たり単価へ反映する。
- 納品準備資材原価は `資材ロット単価 × 使用数量 ÷ 対象商品数量` を商品1単位当たり原価として出荷時に加算する。
- 商品へ直接紐付く外注費は出荷原価内訳へ加算する。
- 出荷確定時に `粗利益 = 売上 - 直接原価`、`粗利率 = 粗利益 ÷ 売上` を保存する。売上0は粗利率未算出とする。
- チップス加工・袋詰め・納品準備の労務費はSKUへ配賦せず、部門共通労務費として工程区分別に集計する。
- 共通フレーバー原料はレシピ／配合マスタで構成を保持できるが、配賦計算はヒアリング確定まで行わない。
- 外注費は会社共通、れんこん部門、青果部門、特定フレーバーの対象区分を保持する。追加の自動配賦ロジックは設けない。

## 重量単位

重量の入力・保存・集計・表示はすべてgとし、kgへの自動換算を行いません。既存kgデータの移行時だけ明示的にgへ変換し、変換元を監査記録へ残します。
- 工程ロス原価 = 廃棄したロット消費原価
- 部門利益 = 売上−材料−包材−労務−外注−製造経費−販管費

資金繰り連携には、売上・仕入の数量原本を渡さず、確定した入出金予定・実績だけを連携します。
