## 移行処理の不足

このコミットの`migratePhase2Schema()`は次だけを扱います。

- T_経費がなければ作成
- T_製造へ使用原料ID・部門IDを末尾追加
- A_買掛へ発生元種別・発生元IDを末尾追加

一方、次は既存DB移行に含まれていません。

- A_工程ロスの新規作成
- T_工程を現在の19列定義へ変換
- T_仕入の空ヘッダーを状態として確定

- g/kg既存値の再計算

したがって、この関数を今から添付Excelへ実行しても、29シート標準との完全一致や工程行の修復は保証されません。本番Excelへ直接実行しません。
